# How can I ensure that my code, program, or software is functional and secure?

This is an easy sounding concept that actually requires the disciplined and iterative approach I reflected on in Project 2.  It takes attention to detail to include every requirement, then testing each requirement to ensure it performs correctly, while rejecting any eronious inputs or outputs. 

# How do I interpret user needs and incorporate them into a program?

User needs are interpreted through meetings and determining requirements or the intent of what the user wants.  AGILE, specifically Scrum, has methodologies for fleshing these out. However, as the developer, I take those requests, needs, structure, etc, communicated by the user/client and build to that specificatio, then test.  The next step in testing is giving that incriment of work back to the user/client for feedback.  

# How do I approach designing software?

I approach designing software through the above principles, and those reflected in my analysis for Project 2. Being methodical, intentional, and taking time to test as I develop, ensure that my designed code is a quality product, usually by finding and correcting errors or bugs early on. If testing my own code, removing as much confirmation bias as I can, write tests designed to fail, and scrutinize my own work with curiosity and objectivity.
